@devify/ui — HTMX Integration

All five HTMX patterns work by adding hx-* attributes to or around dvfy-* components. The components are Light DOM — HTMX can swap them freely without Shadow DOM boundaries. Point the action / src URLs to your own server endpoints.

Direct hx-* attributes on components

Because components use Light DOM, HTMX can observe and process any element. You can place hx-* attributes directly on dvfy-* elements.

Note: HTMX listens for native DOM events. dvfy-* components dispatch standard CustomEvents that bubble, so hx-trigger works out of the box.

CSRF Token Integration

dvfy-htmx-form auto-reads the CSRF token from a <meta name="csrf-token"> tag and injects it as a hidden form field. HTMX itself can also read it via hx-headers or htmx.config.defaultHeaders. 

<!-- Option A: dvfy-htmx-form reads it automatically -->
<meta name="csrf-token" content="{{ .CSRFToken }}">
<dvfy-htmx-form action="/submit">...</dvfy-htmx-form>

<!-- Option B: set globally for all HTMX requests -->
<script>
  document.addEventListener('htmx:configRequest', (e) => {
    e.detail.headers['X-CSRF-Token'] =
      document.querySelector('meta[name="csrf-token"]')?.content;
  });
</script>
